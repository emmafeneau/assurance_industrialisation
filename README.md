# Auto Insurance Pricing Engine

This project builds an auto insurance pricing engine from historical contract and claims data.

The goal is to estimate the expected cost of claims in order to compute a technically justified pure premium, following the classical actuarial decomposition:

```
Pure Premium = Claim Probability × Average Conditional Claim Cost
```

## Usage

### Step 1 — Train the weight imputation model (once)

The severity model uses a Random Forest to impute missing vehicle weights. This pickle must be generated before any training.

```bash
cd Severite/src
python train_rf_poids.py
```

Saves `Severite/models/model_rf_poids.pkl`

---

### Step 2 — Train the frequency model (once)

```bash
cd Frequence/src
python main.py --mode train
```

Saves `Frequence/models/catboost_calibrated.pkl`

---

### Step 3 — Train the severity model (once)

```bash
cd Severite/src
python main.py --mode train
```

Saves `Severite/models/catboost_severite.pkl`

---

### Step 4 — Compute premiums

```bash
cd Prime/src
python main.py

# On a specific file
python main.py --data ../../data/test.csv
```

Exports `Prime/outputs/primes.csv` with `index` and `prime` per contract

---

### Generate individual predictions (optional)

```bash
# Claim probabilities
cd Frequence/src
python main.py --mode predict

# Claim amounts
cd Severite/src
python main.py --mode predict
```

## Models

### Frequency Model

Predicts the probability that a policyholder will have at least one claim during the coverage period.

- **Algorithm**: CatBoostClassifier
- **Calibration**: Isotonic regression on a dedicated calibration set (separate from validation set)
- **Main metric**: Calibration (mean predicted probability ≈ observed claim frequency)
- **AUC**: ~0.63

> **Why is calibration the main metric?**
> The goal is not classification but producing reliable probabilities for fair pricing. Overpriced premiums lose customers; underpriced premiums lose money.

**Data split:**

| Set | Proportion | Role |
|---|---|---|
| Train | 60% | Model training |
| Validation | 20% | Early stopping |
| Calibration | 10% | Isotonic calibration |
| Test | 10% | Final evaluation |

### Severity Model

Predicts the claim amount when a claim occurs. Trained only on contracts with at least one claim.

- **Algorithm**: CatBoostRegressor
- **Metric**: RMSE
- **Imputation**: Random Forest for missing vehicle weights
- **Feature engineering**: Physical ratios and vehicle interactions

## Feature Engineering

### Frequency
- Missing value handling (`sex_conducteur2` → `"Aucun"`, `anciennete_vehicule` → median)
- Removal of collinear variables (r > 0.95) and zero-importance features
- Age discretization into business brackets: 18-30, 31-50, 51-70, 71+
- Department extraction from postal code (Corsica handling: 2A/2B)

### Severity
- Manual anomaly corrections (vehicle weight, engine displacement, price)
- Driver license seniority inconsistency correction
- Hierarchical weight imputation (groupby → Random Forest)
- Derived features: power-to-weight ratio, kinetic energy, vehicle sportiness, price ratios

## API

The REST API is built with FastAPI and exposes the following endpoints:

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/v1/health` | Check API and model status |
| `POST` | `/api/v1/predict/frequence` | Predict claim probability |
| `POST` | `/api/v1/predict/severite` | Predict average claim cost |
| `POST` | `/api/v1/predict/prime` | Compute pure premium |
| `GET` | `/api/v1/vehicles` | List available vehicle brands and models |
| `GET` | `/api/v1/predictions` | Retrieve prediction history |
| `POST` | `/api/v1/explain` | Get SHAP feature importance values |

Interactive documentation available at `/docs`.
